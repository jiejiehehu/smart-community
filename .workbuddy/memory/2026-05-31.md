## 修复：引导弹窗点击无反应

**问题**：点击"开始演示"按钮无任何反应，控制台报 `window.closeGuide is not a function`

**根本原因**：`index.html` 第 2963 行附近有 JS 语法错误（多余的 `}` 和 `});`），导致整个 `<script>` 块执行中断，`window.closeGuide` 从未被定义。

**修复方式**：
1. 删除 DOMContentLoaded 回调中多余的 `}` 和 `});`（第 2960-2965 行）
2. 将 `window.closeGuide` 定义移到页面顶部第一个 `<script>` 标签内，确保按钮 HTML 解析前已定义
3. 按钮 `onclick="window.closeGuide()"` 保持不变

**状态**：✅ 已解决，线上版本正常

## 修复：健康服务 API 404/500 错误

**问题**：H5 端提交健康服务预约报 404，线上 API `/api/health/orders` 返回 500

**根本原因**：
1. 前端提交到 `/api/health/orders`，但后端路由实际是 `/api/health/records`（路径不匹配）
2. `health_orders` 表缺少 `created_at` 列，导致查询失败（500）
3. `health_orders` 的建表语句和 API 端点之前未正确插入 `app.py`

**修复方式**：
1. 修正 `h5.html` 中 `submitHealth()` 的 API 路径：`/api/health/orders` → `/api/health/records`
2. 在 `init_db()` 中添加 `health_orders` 建表语句（含 `created_at` 列）
3. 在 `init_db()` 末尾添加列兼容性处理（`ALTER TABLE` 添加缺失列）
4. 在 `init_mock_data()` 中添加健康服务模拟数据
5. 添加完整的 `health_orders` API 端点（GET/POST/accept/process/complete/review）

**状态**：✅ 已解决，线上 API 正常返回数据
