## Bug修复：H5居民端 JS 字符串拼接语法错误
**根因**：h5.html 第1480行，`startSigning()` onclick 属性使用了错误的字符串拼接：
`'',''` → JS 解析为字符串结束 `'` + 逗号运算符 `,` + 字符串开始 `'`，导致 `SyntaxError: Unexpected string`
整段JS无法执行 → 页面所有功能（showPage/loadService/checkBackend均失效）→ "完全不可用"

**修复**：
- `+ d.name + '','' + d.title +` → `+ d.name + '\', \'' + d.title +`
- 同时修复 `static/h5.html` 和 `deploy/h5.html`
- `node --check` 验证通过后重新部署

## Bug修复：commdoctor.js 与 index.html 的 const API_BASE 重复声明
**根因**：commdoctor.js 和 index.html 都在全局作用域声明 `const API_BASE`，导致 `SyntaxError: Identifier 'API_BASE' has already been declared`，整段 commdoctor.js 无法执行，所有社区医生功能（列表/签约/随访/预警/问诊）全部失效，页面显示永久"加载中..."。

**修复**：从 commdoctor.js 中删除重复的 `const API_BASE` 和 `api()` 函数声明（改为注释说明复用 index.html 中已定义的），同步修复 `static/commdoctor.js` 和 `deploy/commdoctor.js`。
